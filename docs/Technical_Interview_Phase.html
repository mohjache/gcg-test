Technical Interview Phase

Thank you for joining us in the next stage of our interview process.

This project is designed to give you the opportunity to demonstrate how you approach and interpret a brief, make architecture decisions, and deliver working software. There are no trick questions — we're simply interested in how you think and work.

You are encouraged to use AI coding tools such as Cursor or similar as this reflects how we work day-to-day. What we're most interested in is how you direct and apply those tools, not whether you use them.

Please read the brief carefully. Some details are intentionally left open. How you handle ambiguity is part of what we're looking for.

Project Brief

Martin's Movies is a client who reviews movies online. They have moved all of their movies to an online movie database which is only accessible via an API.

They need to:

  • Build a new paginated listing page to pull all data from their new database
  • Allow customers to search by different movie attributes such as Keyword
  • Mark a movie as 'watched' so when they open the listing page again in the same browser it will still be marked as 'watched'

Martin's Movies would like to link the Read More button for each movie off to the IMDB page if the imdb_id is set for a movie.

He would like the design to match a theme he liked, but include a central search input:

https://gnodesign.com/templates/movify/movie-grid3.html

Movie API

For this project, we are using The Movie Database API:

https://developers.themoviedb.org/3/getting-started/introduction

You can sign up for a free account to get an API key.

Deliverables

A ReactJS / NextJS App broken up as you see fit:

  1. Provide the URL to the deployed app
  2. Provide a link to the source code saved to a public GitHub repo
  3. Provide a NOTES.md file in the root of your repo documenting:
    1. How you interpreted the brief and any assumptions you made where it was ambiguous
    2. How you chose to architect the project and why
    3. How you used AI tools (e.g. Cursor, Copilot) during the build — be specific about what you prompted it to do, where it helped, and where you had to correct or override it
    4. Anything you would do differently or add given more time
